### 决策

**agent search vs RAG（v0.1 选 agent search）**

放弃了原计划的 chromadb + BGE-large-zh embedding RAG，改为：
- **检索**：keyword search（rank_bm25 over `title + summary + tags`，纯 python，零部署）
- **agent loop**：LLM 通过 tool_use 多轮调用 `search_regulations(q)` / `get_regulation(id)`；最大 4 轮防失控
- **provider**：v0.1 主推 Anthropic + DeepSeek（用户已配 DeepSeek key），OpenAI schema 兼容免费 work，Ollama tool_use 参差先 punt
- **UI 创新**：前端新增 `AgentTrace.tsx` 折叠面板，把 agent 的 search/get 调用过程展示给用户 —— 比黑盒 RAG 透明

理由：
1. 30–50 条种子规模下 BM25 / keyword 命中已经够；LLM 通过查询重写 + 读完整条款补足检索粗糙
2. 节省一整套 embedding / chunking / vector DB / similarity tuning 工程
3. 演示价值：agent reasoning trace 本身就是 feature，不是噪音
4. v0.5 真做用户、数据超 500 条或上 PDF 多模态时再上 vector DB 不迟（升级路径清晰）

**chromadb 暂不引入**：保持 backend 依赖最少；也避开了 chromadb 在 macOS arm64 上 sqlite 版本相关的安装坑

**tool_use schema 双格式**：tools.py 同时输出 Anthropic JSON Schema 格式和 OpenAI function 格式，runtime 按 provider 选择，避免后续切 provider 重写

### v0.1.0 封版（2026-04-25 末段）

**用户自查发现并修复 2 个 frontend bug（ChatPanel + workspaceStore）**：

1. **`ChatPanel.tsx` infinite re-render**：`deriveConfig(pathname)` 每次返回新对象 → `useChatStream(config)` 的 `useCallback(send)` 依赖变化 → 新 send 身份 → `pendingQuery` effect 死循环。

   - 修复：模块级 `EMPTY_CARDS` / `EMPTY_TRACE` 常量 + selector 全部 null-safe（接受 `moduleId | null`）（[workspaceStore.ts:147-194](frontend/src/stores/workspaceStore.ts#L147-L194)）
