## 范围 + Done 定义

### 用户能做（v0.2 release 后）

在 `/bookkeeping` 页面拖入 1 张发票（jpg / png / pdf）+ 写一句业务说明 → 后端：
1. OCR 抽字段 → `InvoiceParsed`（卖方 / 买方 / 明细 / 不含税金额 / 税额 / 总额 / 票号 / 日期）
2. BookkeepingAgent 多轮 tool_use：先看发票字段 → 调 v0.1 `search_regulations` 找适用准则 → `get_regulation` 取条款全文
3. 流式输出 `VoucherDraftCard`（借/贷/科目/摘要/金额，**前端可编辑**）+ `TaxRiskCard`（可抵扣判断 + 限额提示）+ 复用 v0.1 的 `RegulationSnippetCard`

右栏 chat 可基于当前 voucher 状态追问（"为啥借方记业务招待费而不是差旅费？"），agent 知道用户已编辑过的最新 voucher。

### 不做

- 银行流水批量识别（v0.3 月结需要时再做）
- 多张发票批量上传（v0.2 单张演示足够）
- 凭证持久化（v0.4 SQLite 才入库；v0.2 编辑只在 zustand，刷新即丢，UI 加红条提示）
- 税号校验 / 重复入账检测（v0.3 月结模块负责）

### Release 标准

- ≥ 50 backend tests 全绿（OCR adapter mock + agent flow + 编辑流端到端）
- 3 类发票实测端到端跑通：增值税专用发票 / 普通发票 / 餐饮票
- `npm run build` 通过，`/bookkeeping` 路由 first-load 不超过 130 KB（dropzone 增量预算）
- `git tag v0.2.0` + `docs/versions/v0.2.0-开发中.md` → `git mv` → `v0.2.0-封版.md`

## 技术依赖

### OCR provider 决策（PRD §9 决策 C，D1 必锁）

**推荐**：腾讯云 OCR `VatInvoiceOCR` / `RecognizeGeneralInvoice` 主用 + GPT-4o vision fallback

| 选项 | 准确率 | 成本 | 易用性 |
|---|---|---|---|
| 腾讯云 OCR | 高 | $（几分钱/张） | API 易用，增值税专票模型现成 |
| GPT-4o vision | 高 | $$$ | 灵活但贵；标准发票格式不必要 |
| Azure Document Intelligence | 高 | $$ | 中文增值税票不如腾讯精准 |
| 自部署 PaddleOCR | 中 | 免费 | 部署麻烦 |

**fallback**：env 没配腾讯 key 时切到 GPT-4o vision —— 保证开发者无需注册腾讯账号也能跑 demo。

### 新依赖

- pip：`tencentcloud-sdk-python-ocr>=3.0.1100` / `python-multipart>=0.0.9`（FastAPI 文件上传）/ `Pillow>=10.0`（EXIF 旋转 + 压缩）
- 前端：`react-dropzone@^14`
- env：`TENCENT_OCR_SECRET_ID` / `TENCENT_OCR_SECRET_KEY` / `TENCENT_OCR_REGION=ap-guangzhou` / `OCR_FALLBACK_PROVIDER=gpt4o`
